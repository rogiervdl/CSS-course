Basic CSS rule

  • A basic CSS rule consists of a selector, a property and a value: 
    selector {
  property: value;
}
  • Example: 
    p {
  color: black;
}
  • → this selects all paragraphs, and sets the text color to black

CSS

Selectors basics

Tag selector

  • A tag selector selects elements by tag name: 
    p { /* all paragraphs */
  ...
}

Id selector

  • An id selector selects a single element by its id attribute: 
    #btnLogin { /* element with id="btnLogin" */
  ...
}

Class selector

  • A class selector selects elements by class attribute: 
    .message { /* elements with class="message" */
  ...
}
  • You can combine with other selectors: 
    li.alternate { /* all list items with class="alternate" */
  ...
}
#dialog1.open { /* element with id="dialog1" and class="open" */
  ...
}
a.button.green { /* all links with class="button green" */
  ...
}

Pseudo class selector

  • A pseudo class selector selects elements in a certain state or relation: 
    :focus { /* any element that has focus */ }
:first-child  { /* all elements that are the first child of their parent */ }
  • Some popular pseudo class selectors combined with tag selectors: 
    a:visited { /* link is visited */ }
a:hover { /* link is currently hovered */ }
a:active { /* link is currently active */ }
a:focus { /* link has currently focus */ }

li:first-child  { /* select first li */ }
tr:nth-of-type(3n+1)  { /* select rows 1, 4, 7 ... */ }

Pseudo element selector (1)

  • A pseudo element selector selects a part of an element: 
    ::selection { /* cursor selected content */ }
  • Some popular pseudo element selectors combined with tag selectors: 
    em::first-letter { /* first letter of emphasize element */ }
p::first-line { /* first line of paragraph */ }
Actually, the double colon notation was introduced in CSS3 to make a differentiation between pseudo class and pseudo element selectors. For the sake of really old browsers who do not support this, you can still use the old single colon notation: 
em:first-letter { /* ... */ }
p:first-line { /* ... */ }
We will however always use the double colon notation.

Pseudo element selector (2)

  • Inject content before or after an element with ::before and ::after: 
    <!doctype html>
<html>
<head>
  <title>testpage</title>
  <meta charset="utf-8" />
  <style>
    li {
      list-style: none; /* remove standard bullets */
    }
    li::before {
      content: "👉";
      font-size: 12px;
      margin-right: 5px;
    }
    li::after {
      content: "👈";
      font-size: 12px;
      margin-left: 5px;
    }
  </style>
</head>
<body>
  <ul>
    <li>item 1</li>
    <li>item 2</li>
    <li>item 3</li>
  </ul>
</body>
</html>

note: you could also use the list-style-image property to set an image as bullet, but then you won't be able to style your bullet with margins, font sizes etc... like with ::after

Contextual selector (1)

  • A contextual selector selects elements based on their position relative to other elements
  • Element nested inside another parent element (read from right to left): 
    #leftcol p { /* paragraphs in element with id="leftcol" */
  ...
}
nav li.active a { /* links in list items with class="active" in nav element */
  ...
}

Contextual selector (2)

  • Element directly nested inside another element: 
    header > p { /* styles for <p>'s where <header> is direct parent */
  ...
}
  • Element directly following another element: 
    h2 + p { /* styles for <p>'s directly following a <h2> */
  ...
}

Grouping selectors

  • Different selectors may be grouped in a comma-separated list: 
    input, #mainmenu a:hover, p.formpar {
  color: blue;
}
  • This is equal to: 
    input {
  color: blue;
}
#mainmenu a:hover {
  color: blue;
}
p.formpar {
  color: blue;
}

Properties basics

Property list

  • Set selector styles with a list of property - value pairs: 
    selector {
  property 1: value 1;
  property 2: value 2;
  property 3: value 3;
  ...
}
    p {
  font-size: 10px;
  color: red;
  margin-bottom: 6px;
}

Grouping properties (1)

  • Some properties may be grouped in a single values list, e.g... 
    body {
  background-image: url(../img/bg.jpg);
  background-repeat: repeat-y;
  background-position: right;
}
.box {
  margin-top: 6px;
  margin-right: 15px;
  margin-bottom: 6px;
  margin-left: 15px;
}

Grouping properties (2)

  • ...is the same as: 
    body {
  background: url(../img/bg.jpg) repeat-y right; /* image, repeat, position */
}
.box {
  margin: 6px 150x 6px 150x; /* top, right, bottom, left */
}
  • → the order of property values is important!
    → personally, I don't group properties often (too hard to read and edit)

Quotes around properties

  • Put quotes around values with spaces: 
    p {
  font-family: Arial, Verdana, "sans serif";
}
  • Don't put quotes around other values: 
    body {
  background-image: url(img/bg.jpg);
}

Priority & Inheritance

  • Final style of an element is a combination of
    • styles can be defined in multiple CSS files
    • some styles (mostly font and visibility) are inherited from the parent
    • if a style isn't defined, browser defaults are used
  • In case of conflicting styles, priority is determined by
    • selector weight
    • in case of same weight: last one wins
  • Inline styles (style="...") always have highest priority

Priority by order (1)

  • Example: 
    p {
  font: bold smaller Arial;
  font-family: Courier, Verdana;
}
  • → conflict between Arial and Courier
  • → selector is the same for both properties
  • → last one wins, which is Courier

Priority by order (2)

  • What is wrong with this fragment? 
    a:active {
  color: white;
}
a:hover {
  color: lightblue;
}
  • → active is also hover, so link will be lightblue when active
  • → solution: switch both fragments

Priority by weight (1)

  • Example: 
    div#idtest {
  font-size: 12px;
}
div.ctest {
  font-size: 14px;
}
  • This element is selected by both selectors: 
    <div id="idtest" class="ctest">...</div>
  • → the first selector weighs more, so 12px

Priority by weight (2)

  • Calculating the specificity (weight) of a selector is complex
  • Generally, from heavy to light:
    • heavy: id selectors
    • normal: class-, attribute-, and pseudo-class selectors
    • light: tag selectors
    • ignored: universal selector, pseudo-element selectors...

Inheritance

  • Some styles are inherited by elements from their parent, mostly visibility and fonts (see list on this page), example: 
    body {
  color: #00b;
  font-family: courier;
  border: 5px solid black;
}

    ...
<body>
  <h2>This is line 1</h2>
  <p>This is line 2</p>
</body>
</html>
    demo
  • → <h2> and <p> inherit color and font from <body>, but not border

Advanced selectors

Universal selector

  • The universal selector selects all elements: 
    * { /* all elements */
  ...
}
  • The universal selector is slow, use inheritance instead if you can, e.g. 
    * { 
  font-size: 12px; /* applied to all elements = slow */
}
    body { 
  font-size: 12px; /* applied to one element = fast; inherited by all children */
}

Attribute selector

  • An attribute selector selects elements by attribute: 
    select[multiple] { ... }    /* has, e.g. <select multiple> */
input[type=button] { ... }  /* is, e.g. <input type="button"> */
a[title*=flower] { ... }    /* contains, e.g. <a title="...flower..."> */
img[src^=https://] { ... }  /* starts with, e.g. <img src="https://..."> */
a[href$=".pdf"] { ... }     /* ends with, e.g. <a href="....pdf"> */

More pseudo classes (1)

  • Structural pseudo class selectors... 
    p:first-child { ... }   /* first child, if it is a paragraph */
p:last-child { ... }    /* last child, if it is a paragraph */
p:first-of-type { ... } /* first paragraph */
p:last-of-type { ... }  /* last paragraph */
p:nth-of-type(3n+1)     /* paragraphs 1, 4, 7... */
p:nth-child(4n+2)       /* child 2, 6, 10..., if it is a paragraph */

More pseudo classes (2)

  • ...and form pseudo class selectors... 
    input:enabled { ... }   /* enabled input */
input:disabled { ... }  /* disabled input */
input:checked { ... }   /* checked input */
input:required { ... }  /* required input; same as input[required] */
input:valid { ... }     /* valid input */
input:invalid { ... }   /* invalid input */

[More pseudo classes (3)]

  • ...and more pseudo class selectors: 
    :only-of-type { ... }  /* element unique in its parent */
:only-child { ... }    /* element that is only child of its parent */
:empty { ... }         /* empty element */
:target { ... }        /* is target of current link */
:not(.p) { ... }        /* anything which is not a paragraph */

CSS tips

tip 1: use CSS reset

  • Start with a CSS reset to wipe out undesired browser defaults.
  • Many exist, but we'll use this one in the course (details explained later): 
    html {
  font-size: 62.5%;
}
body {
  font-size: 1.4rem; /* change this to your default font size */
}
* {
  box-sizing: border-box;
}
img {
  max-width: 100%;
}
  • → include this at the top of your CSS

Tip 2: limit selector length

  • Keep your selectors as short as possible. 
  • body header > nav.top ul li a {
  ...
}
div#wrapper {
  ...
}

    header > nav a {
  ...
}
#wrapper {
  ...
}
  • → better readability
    → lower specificity = easier to overrule later with stronger selector

Tip 3: sort your properties

  • Sorting your properties list alphabetically helps readability: 
  • .some-element {
  /* don't worry about the CSS itself; note how properties are sorted a-z */                
  background-color: #A2C355;
  border: 2px solid #A2C355;
  border-radius: 2px;
  color: #fff;
  cursor: pointer;
  display: inline-block;
  font-size: 1.4rem;
  line-height: 1.8rem;
  padding: 0.9rem 2.4rem;
  text-decoration: none;
  text-transform: uppercase;
  transition: background 0.15s, color 0.15s ease-in-out;
}

[Tip 4: use BEM (3)]

  • Most CSS is written this way, and that's okay.
  • It has some drawbacks though: 
    main > section { /* if you insert a div in your HTML, your CSS breaks */
  ...
}
#products { /* where do we find this element? main content, sidebar...? */
  ...
}
.product h4 { /* if we use a h5 instead, our CSS breaks */
  ...
}
.description { /* too general; other elements might have class="description" */
  ...  
}
#products .visual + p { /* specific enough, but long long and heavy selector */
  ...
}

[Tip 4: use BEM (4)]

  • Most CSS suffer from these problems:
    • the CSS depends too much on the HTML structure
    • the selectors lack meaning and context
    • the selectors are not unique enough, and may affect other elements
    • if selectors are specific enough, they tend to become long, heavy and complex
  • Consider using the BEM (Block-Element-Modifier) technique:
    • always use class selectors by default; avoid other selectors like id, tag, attribute, ...
    • use the block__element--modifier naming concention:
      • block: the element's parent
      • element: description of the element
      • modifier (optional): state (e.g. open/closed) or theme (e.g. green/blue)
