Units

Overview

Unit Type Related to Example

px (1)

  • pi(cture) el(ement).
  • on normal screens = 1 real pixel, on other devices complicated formula:

So the pixel is actually an angle, not a size. The moon is 24.3px, for example.

A screen pixel is defined as the angle at 71cm distance, being the average arm length of a user. I kid you not.

px (2)

  • In CSS px is used for sizes that should not scale or scale stepwise, like borders and small icons: 
    input[type=submit] {
  border: 2px solid #666;
}
img.icon_social {
  width: 22px;
  heigt: 22px;
}
  • although you can use px everywhere (and many still do), it will make the step to responsive CSS much harder

small images like icons may become ugly when scaled continuously

Em (3)

  • to prevent overlap, the line height should be > 1em (default 1.2em)
Most browsers will devide the line gap in two, so to have a text touch the top or bottom of a box, you'll have to nudge it a little with position: relative.

Em (4)

  • In CSS, em means percentage of parent font size

    footer { font-size: 0.8em; /* size: 0.8 * 14px = 11.2px */ }
    footer a { font-size: 1.1em; /* size: 1.1 * 11.2px = 12.32px */ }
  </style>
</head>
<body>
  <h1><a href="#">My page</a></h1>
  <p>this is the second paragraph</p>
  <footer>also read <a href="#">this</a> </footer>
</body>
</html>

Rem (1)

  • The rem (root em) is like em but relative to the <html> element.
  • Just devide all px values by 16: 
    <body>
<head>
  <style>
    body { font-size: 0.875rem; /* = 14/16 */ }
    h1 { font-size: 1.25rem; /* = 20/16 */ }
    footer { font-size: 0.6875rem; /* = 11/16 */ }
    footer a { font-size: 0.75rem; /* = 12/16 */ }
  </style>
</head>
<body>
  <h1><a href="#">My page</a></h1>
  <p>this is the second paragraph</p>
  <footer>also read <a href="#">this</a> </footer>
</body>
</html>
  • This leads to easier calculations and slightly easier numbers, but still not very intuitive though...

Rem (2)

  • The trick is to change the font size of the html to 10px: 
    <body>
<head>
  <style>
    html { font-size: 62.5%; } /* 16px * 0.625 = 10px */
    body { font-size: 1.4rem; /* = 1.4 * 10px */ }
    h1 { font-size: 2rem; /* = 2 * 10px */ }
    footer { font-size: 1.1rem; /* = 1.1 * 10px */ }
    footer a { font-size: 1.2rem; /* = 1.2 * 10px */ }
  </style>
</head>
<body>
  <h1><a href="#">My page</a></h1>
  <p>this is the second paragraph</p>
  <footer>also read <a href="#">this</a> </footer>
</body>
</html>
  • Now you can just devide all pixel values by 10

Rem (3)

  • Recently introduced in CSS3, but supported by all major browsers and IE9+.
  • As easy as px, but we can still rescale the whole page by changing the 62.5% (try it out)
  • For small values that should not scale or scale stepwise, you should still use px
  • For independent components, you may still consider em

percentage %

  • For fonts it is identical to em: 
    p {
  font-size: 80%; /* same as 0.8em */
}
  • For width and height it refers to the parent dimensions: 
    img {
  width: 80%; /* 80% of the parent's width */
  max-width: 600px;
}
    demo
  • to avoid confusion, we'll use em for fonts, and % for dimensions that scale with device width

vw / vh / vmin

  • Percentage of viewport (= visible browser area) width / height / minimum of both
  • Example 1: font rescaling (also used in this presentation; try rescaling the browser): 
    p {
  font-size: 2vw; /* Â± 80 characters per line */
}
    (slightly different) demo
  • Example 2: keeping an image visible 
    img {
  max-width: 90vw;
  max-height: 90vh;
}
    demo
  • vw/vh/vmin is the only way to smoothly rescale fonts

Best practices

  • When to use which unit:
    • by default, use rem
    • consider em for self-contained components like widgets
    • use vw/vh for dimensions you'd like to scale with device width or height
    • use % for dimensions you'd like to scale with parent width
    • use px for small values that scale bad, like border widths or small icons

  • .box {

  padding: 20px;

Vendor prefixes

  • CSS3 experimental features lshould be prefixed to allow browser vendors to experiment with their own implementation. You might run across code like this: 
  • .box {
  -webkit-box-sizing: content-box; /* Chrome and Safari */
  -moz-box-sizing: content-box; /* Firefox */
  -o-box-sizing: content-box; /* Opera */
  -ms-box-sizing: content-box; /* Internet Explorer */
  box-sizing: content-box; /* generic */
}
  • Once consensus is reached on the implementation, the browser prefixes can be omitted. Actualized list see shouldiprefix.com/

Hacks and filters

IE5 box model hack

  • Hacks are an old-fashioned way to solve poor browser support
  • Mother of all hacks: IE5 box model hack 
    #pup {
  border: 20px solid;
  padding: 1px;
  width: 250px; /* voor IE5/Win */
  voice-family: "\"}\"";
  voice-family: inherit;
  width: 208px; /* voor alle andere browsers: */
}
/* be nice to Opera 5 */
html > body #pup {
  width: 208px;
}
  • Fixes IE5 box model misinterpretation
  • IE5 and any IE version < 10 can safely be ignored these days

IE5/Win escape filter

  • IE5/Win escaped property filter 
    #content {
  w\idth: 100px; /* ignored by IE5/Win */
}
  • IE6/Win and lower underscore filter 
    #content {
  position: fixed;
  _position: static; /* only executed by IE6/Win and lower */
}
  • IE6/Win and lower child selector filter 
    html > body {
  background-image: url(bg.png);
  /* not executed by IE6/Win and lower */
}

Just don't

  • Hacks and filters had their role in early CSS times
  • These days: avoid using them at all times
  • Use conditional CSS for IE instead if necessary

Image replacement

  • Replace textual elements like headers, links etc... with graphics: 
    <h1><a href="#">Your Logo Here</a></h1>

    h1 a {
  display: block; /* only block level elements can have width / height */
  width: 21.4rem; /* stretch link to desired width */
  height: 16.1rem; /* stretch link to desired height */
  white-space: nowrap; /* single line of text */
  text-indent: 100%; /* push text outside */
  overflow: hidden;  /* hide text outside */
  background-image: url(img/logo.jpg); /* set background image */
  background-size: 100% 100%; /* stretch background image */
}
