Units

Overview

Units used in this course:

Unit Type Related to Example
px absolute screen pixels p {font-size: 14px}
em percentage parent element font size p {font-size: 1.2em}
rem percentage root element font size p {font-size: 0.75rem}
% percentage parent dimension or font size .left {width: 20%}
  • more advanced units (not used in this course): vmin, vmax, vw, vh
  • even more units exist (ex, pt, pc, cm, in...) but none is useful in websites

[px — definition]

  • pi(cture) el(ement).
  • on normal screens = 1 real pixel, on other devices complicated formula:

So the pixel is actually an angle, not a size. The moon is 24.3px, for example.

A screen pixel is defined as the angle at 71cm distance, being the average arm length of a user. I kid you not.

px — when to use

  • px is a fixed unit, used for sizes that do not scale, like borders and small icons: 
    input[type=submit] {
  border: 2px solid #666;
}
img.icon_social {
  width: 22px;
  heigt: 22px;
}
  • use px for fixed sizes

Small images like icons may become ugly when scaled continuously, so there using px makes sense

Note: although some use pixels everywhere, this makes responsive CSS much harder. For anything that should shrink or grow depending on screen size, use other units, like em, rem or %

[Em — definition (1)]

  • in ancient times defined as the height of metal letter bodies, which was square for the letter M:

[Em — definition (2)]

  • in digital fonts with serifs (the little ornaments at the end of letters), the actual height may be a little more than 1em:

[Em — definition (3)]

  • to prevent overlap, the line height should be > 1em (default 1.2em)
Most browsers will devide the line gap in two, so to have a text touch the top or bottom of a box, you'll have to nudge it a little with position: relative.

Em — definition (4)

  • in CSS, em means percentage of parent font size
  • to know the px value, multiply with the parent font size or browser default: 
    <html>
<head>
  <style>    
    html { font-size: 1em; } /* 1 x browser default font size = 16px */
    body { font-size: 0.875em; /* 0.875 x html font size = 14px */ }
    h1 { font-size: 1.4em; /* 1.4 x body font size = 19.6px */ }
    article { 
      width: 30em; /* 30 x body font size = 420px */ 
      font-size: 1.2em; /* 1.2 x body font size = 16.8px */
    }
    a { font-size: 1.1em; /* 1.1 x parent font size */ }
  </style>
</head>
<body>
  <h1>Articles about <a href="#">this topic</a></h1>

</html>

  • for <html>, the reference is the default font size set the browser, which usually 16px, see settings:

em — when to use (1)

  • em is a relative unit, used for components that scale with parent font size, e.g. widgets or other independent components
  • try changing the widget container font size in this demo (and note how em is used everywhere in the widget):

demo

em — when to use (2)

  • em has one distinct advantage:
    • you can rescale a whole component by changing the parent font size
  • em has a few serious drawbacks:
    • the final size is the result of a chain of calculations, which can quickly become complicated
    • the numbers you use aren't very intuitive
    • most designers still think in pixel sizes, not in relative units
  • I use em rarely; mostly for widgets

% — definition

  • used for setting width and height as a percentage of the parent dimensions: 
    img {
  width: 80%; /* 80% of the parent's width */
  max-width: 600px;
}
    demo

% — when to use

  • % is also a relative unit
  • used for components that scale with parent width, e.g. columns, images...

Rem — definition (1)

  • rem or “root em” is like em but relative to the <html> element
  • to know the px value, multiply with the <html> font size: 
    <html>
<head>
  <style>
    html { font-size: 1em; /* 1 x browser default font size, usually 16px */ }
    body { font-size: 0.875rem; /* 0.875 * 16px = 14px */ }
    h1 { font-size: 1.25rem; /* 1.25rem * 16px = 20px */ }
    footer { font-size: 0.6875rem; /* 0.6875 * 16px = 11px */ }
    footer a { font-size: 0.75rem; /* 0.75 * 16px = 12px */ }
  </style>
</head>
<body>
  <h1><a href="#">My page</a></h1>
  <p>this is the second paragraph</p>
  <footer>also read <a href="#">this</a> </footer>
</body>
</html>
  • calculations are very easy: px = rem * 16

Rem — definition (2)

  • it becomes even easier if we set <html> font size to 62.5%: 
    <html>
<head>
  <style>
    html { font-size: 62.5%; } /* 16px * 0.625 = 10px */
    body { font-size: 1.4rem; } /* 1.4 * 10px = 14px */ 
    h1 { font-size: 2rem; } /* 2.0 * 10px = 20px */
    footer { font-size: 1.1rem; } /* 1.1 * 10px = 11px */
    footer a { font-size: 1.2rem; } /* 1.2 * 10px = 12px */
  </style>
</head>
<body>
  <h1><a href="#">My page</a></h1>
  <p>this is the second paragraph</p>
  <footer>also read <a href="#">this</a> </footer>
</body>
</html>
  • calculations are even easier: px = rem * 10!

Rem — definition (3)

  • bottomline: always start your site CSS with 
    html { 
  font-size: 62.5%; /* set HTML font size to 62.5%; now 1rem = 10px */
} 
body { 
  font-size: 1.4rem; /* set default font size, e.g. to 14px */ 
}

/* rest of your CSS follows here */
/* ... */

Rem — when to use

  • rem is both fixed (independent of parent) and relative (to <html> font size)
  • use rem as your default unit except where px, or % is needed
  • as intuitive as px, but we can still rescale the whole page by changing the <html> font size (try it out — use your inspector)

[vw / vh / vmin / vmax]

  • Percentage of viewport (= visible browser area) width / height or minimum / maximum of both
  • Example 1: font rescaling (also used in this course; rescale the browser): 
    p {
  font-size: 2vw; /* Â± 80 characters per line */
}
    (slightly different) demo
  • Example 2: keeping an image visible 
    img {
  max-width: 90vw;
  max-height: 90vh;
}
    demo
  • mostly useful for mobile devices
  • vw/vh/vmin/vmax is the only way to smoothly rescale fonts

Units — summary

  • When to use which unit:
    • by default, use rem; set html font size to 0.625rem first, so that 10px = 1rem
    • consider em for self-contained components like widgets
    • use % for dimensions you'd like to scale with parent width like columns
    • use px for small values that scale bad, like border widths or small icons

What is a font

  • A font is a graphical representation of text. Properties:
    • font-family: Arial, Courier New, Times New Roman...
    • font-size: 8px, 12px, 18px...
    • font-weight: lighter, normal, bold...; 100, 300, 500, 700, 900

  • Definitions (from outside to inside):
    • the margin is the spacing outside an element

Margin bleeding (1)

  • If it doesn't meet a border or padding on its way, a vertical margin will “bleed” out of its parent. Here, the <p> top margin bleeds out of the <header> and pushes it down: 
    <html>
<head>
  <style>
    * { margin: 0; }
    header {
      height: 110px;
      background-color: #ccc;
    }
    #login { margin: 20px; }
  </style>
</head>
<body>
  <header>
    <p id="login"><a href="#">klantenzone</a></p>
  </header>
</body>
</html>

Margin bleeding (2)

  • Solve it by replacing the <p> top margin with a <header> top padding:

Margin & padding notes (1)

  • Margins & paddings may behave strange:
    1. unpositioned blocklevel elements: horizontal margins add up, but for vertical margins, only the largest value counts
    2. positioned blocklevel elements: all margins add up
    3. inline elements: vertical margins are ignored; vertical paddings overlap other content above but underlap below
  • → see this summary demo

Margin & padding notes (2)

  • You don't need to remember all paddings and margins rules. Just avoid a lot of troubles by following these guidelines:
    • don't set vertical paddings or vertical margins on inline elements
    • avoid bleeding margins by using paddings instead where you can
    • above all: use your chrome inspector all the time!

The box model

Box model

  • The model we saw earlier is also referred to as the box model
  • By default, the width and height properties refer to the content only, excluding the borders and paddings. The actual width or height in the below example is not 150px, but 162px:
div {
  width: 150px; /* total width is 162px */
  padding: 5px;
  border: 1px solid black;   
}

Box model problems

  • This model leads to some problems. Let's create a simple horizontal menu of 100% wide, with 4 links 25% wide each: 
    <html>
<head>
  <style>

    ul {
      border: 3px solid black;
      width: 100%;
    }
    a {
      padding: 5px 10px;
      width: 25%;
    }
    li + li a {
      border-left: 1px solid black;
    }

    /* other CSS, not important for this slide */

      border: 1px dotted;
      width: 400px;
    }  
    ul {
      background-color: #ddd;
      list-style: none;
      overflow: hidden; /* layout stuff; you'll learn more later */
      padding: 0;
    }
    a {
      display: block; /* make block level, or setting width won't work */
      background-color: #ccc;
      float: left; /* layout stuff; you'll learn more later */
